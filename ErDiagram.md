# ER Diagram - FinGuard AI

## Database Schema Overview (PostgreSQL + Prisma)
The schema is designed for transaction lifecycle processing, fraud/risk evaluation, alert handling, and immutable auditing. Prisma ORM maps these tables to strongly typed TypeScript models for repository implementations.

## Key Tables
- `users`: identity and account owner profile.
- `roles`, `user_roles`: JWT role authorization model.
- `accounts`: account state and balances.
- `transactions`: financial events and processing outcome.
- `risk_assessments`: score, level, decision, reason metadata.
- `fraud_signals`: atomic signals generated by rules/models.
- `behavior_profiles`: rolling behavioral baseline per user.
- `alerts`: analyst/admin investigation queue.
- `notifications`: delivery status by channel.
- `audit_logs`: append-only compliance trail.
- `fraud_rules`: configurable rule engine definitions.
- `admin_actions`: manual overrides and governance actions.

## Constraints and Data Rules
- Unique:
  - `users.email`
  - `accounts.account_number`
  - `transactions.idempotency_key`
  - `fraud_rules.rule_name`
- Foreign keys with indexed columns for query performance.
- `transactions.amount > 0` check constraint.
- Enumerated status values enforced by app-level enums and DB constraints.
- `audit_logs` append-only policy (no update/delete in service layer).
- Soft delete (`deleted_at`) only for metadata tables; financial events remain immutable.

## Prisma Model Mapping (Example)
- `users` -> `model User`
- `accounts` -> `model Account`
- `transactions` -> `model Transaction`
- `risk_assessments` -> `model RiskAssessment`
- `alerts` -> `model Alert`
- `audit_logs` -> `model AuditLog`

## Indexing Recommendations
- `transactions(account_id, initiated_at desc)`
- `transactions(status, decision, processed_at desc)`
- `risk_assessments(risk_score desc, assessed_at desc)`
- `alerts(status, severity, created_at desc)`
- `audit_logs(correlation_id)`
- `fraud_signals(transaction_id, triggered)`

## Retention and Compliance
- Audit logs retained per regulatory requirement (e.g., 7 years).
- Transactions and risk assessments archived with immutable snapshots.
- Sensitive values masked or encrypted in logs and exports.
